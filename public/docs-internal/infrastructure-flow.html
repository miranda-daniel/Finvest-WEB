⚡ Vite dev server
Runs via npm run dev on port 5100. Serves React source files directly with hot module replacement — saving a file reflects instantly in the browser without a full reload. Also acts as a reverse proxy for /api/*, /logos/*, /posthog/*, and /sentry/*, so the browser never makes cross-origin requests.
🟢 Finvest-API
Node.js process on port 3001. Runs on the host machine or via docker-compose.dev.yml. Handles all REST and GraphQL business logic. Not exposed to the browser directly — always reached through the Vite proxy at /api/*.
🗄️ PostgreSQL
Runs in Docker via docker-compose.dev.yml. Port 5432 exposed to the host so the API can connect. Data is persisted in a Docker volume.
🔴 Redis
Runs in Docker via docker-compose.dev.yml. Port 6379 exposed. Used as an in-memory cache for market data: live quotes (60s TTL) and EOD prices (keyed by date — fetched once per day per symbol set). Prevents hitting TwelveData's rate limit on repeated page loads.
🌍 FMP CDN (logo images)
External service at financialmodelingprep.com/image-stock. Provides stock logo images. Requests are proxied through Vite so the browser sees its own origin — avoids ERR_BLOCKED_BY_ORB caused by missing CORS headers on the FMP side.

☁️ AWS ALB
Application Load Balancer — the public entry point. Handles: SSL/TLS termination (so nginx doesn't need certificates), DNS routing, load balancing across multiple instances, and AWS health checks. Forwards decrypted HTTP traffic to nginx on port 80 inside the VPC.
🔷 nginx
The app-level entry point inside Docker. Five jobs: 1) serves the React dist/ static build for all /* requests, 2) proxies /logos/* to FMP, 3) proxies /api/* to the API container via Docker internal DNS (api resolves automatically), 4) proxies /posthog/* to PostHog (avoids ad-blocker blocks), 5) proxies /sentry/* to Sentry ingest (avoids ad-blocker blocks).
🟢 Finvest-API container
Same Node.js app as in dev, containerized. Not publicly exposed — only reachable internally via nginx. Docker Compose internal DNS resolves api to this container automatically.
⚡ What happened to Vite?
npm run build compiles all React/TypeScript into plain HTML + CSS + JS inside dist/. After that, Vite has zero runtime role. nginx serves those static files directly — no Node.js process needed for the frontend.
