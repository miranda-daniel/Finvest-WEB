# API layer conventions

Stack: Apollo Client + GraphQL + graphql-codegen.

## Folder layout

```
src/api/
  operations/          # .graphql files per domain
    auth/
      login.mutation.graphql
      me.query.graphql
    transactions/
      getTransactions.query.graphql
      createTransaction.mutation.graphql
  hooks/               # custom hooks wrapping Apollo per domain
    auth/
      useLogin.ts
      useCurrentUser.ts
    transactions/
      useTransactions.ts
      useCreateTransaction.ts
  fragments/           # reusable GraphQL fragments
    transaction.fragment.graphql
    user.fragment.graphql
  generated/           # ⚠ DO NOT EDIT — auto-generated by graphql-codegen
    graphql.ts
```

## Core rules

- Components never import `useQuery` or `useMutation` from Apollo directly — always go through hooks in `hooks/`.
- Never import from `generated/` in components — go through hooks.
- Input validation and user-readable errors live inside the custom hook, not in the component.
- Run `npm run codegen` after modifying any `.graphql` file to keep `generated/` in sync.

## Types

- All entity and operation types come from `generated/graphql.ts` — do not write them manually.
- Write manual types **only** for things the schema does not cover: form DTOs, local UI state shapes.
- Place manual types in `src/api/types/{domain}.ts` with a comment clarifying they are not generated.

## Naming — GraphQL operations

| Type | Pattern | Examples |
|------|---------|----------|
| Query | `{Verb}{Descriptor}` | `GetTransactions`, `GetTransactionById`, `GetCurrentUser` |
| Mutation | `{Verb}{Descriptor}` | `CreateTransaction`, `UpdateTransaction`, `Login` |
| Fragment | `{Entity}Fields` | `TransactionFields`, `UserFields` |

- Use `Get` for all queries. Stay consistent — do not mix `Get`, `Fetch`, `List`.
- The operation name must match the filename: `createTransaction.mutation.graphql` → `mutation CreateTransaction`.

## Naming — custom hooks

| Role | Pattern | Examples |
|------|---------|----------|
| Query hook | `use{Descriptor}` | `useTransactions`, `useCurrentUser` |
| Mutation hook | `use{Verb}{Descriptor}` | `useCreateTransaction`, `useLogin` |

Hook names describe **UI intent**, not the underlying GraphQL operation name.

## Examples

```graphql
# operations/transactions/getTransactions.query.graphql
query GetTransactions($filters: TransactionFiltersInput) {
  transactions(filters: $filters) {
    ...TransactionFields
  }
}
```

```ts
// hooks/transactions/useTransactions.ts
export function useTransactions(filters?: TransactionFiltersInput) {
  return useQuery(GetTransactionsDocument, { variables: { filters } })
}
```

```graphql
# fragments/transaction.fragment.graphql
fragment TransactionFields on Transaction {
  id
  amount
  description
  date
  category
}
```
